Step 2
Tap Stack To List

左下スタックをタップして文書一覧へ戻る。文書一覧サムネイルは常に先頭ページ A の small preview を参照する。

Step 3
Tap Document

文書一覧の当該文書をタップしてページ一覧へ進む。large preview が無いページはその場で spinner を出す。

Step 4
Large Ready

ページ一覧の中で large preview が ready のページから順に表示される。全ページ ready になった時点で通常表示へ揃う。

Reading Guide

  • カメラ画面は `/mockups/camera-scan.html` をそのまま表示し、capture・stack・viewfinder の見た目を画面設計と揃える。
  • source image はシャッター直後に保存済みとみなし、small / large preview のみが時間差で ready になる。
  • 文書一覧は A の small preview だけを参照するので、B / C の遅れは一覧サムネイルには現れない。
  • ページ一覧では large preview が page 単位で ready / processing に分かれ、同じ画面に混在して見える。

Design Notes

  • preview worker は idempotent にして、アプリ再起動や途中中断後も安全に再実行できるようにする。
  • 撮り直し時は、その page の source / small / large / working をまとめて invalidation する。
  • camera stack と文書一覧の両方が small preview を使うが、参照ページは最新 C と先頭 A で異なる。
  • ページ一覧の placeholder / spinner は page 単位で残し、画面全体は block しない。
